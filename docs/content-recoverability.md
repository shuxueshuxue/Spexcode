# content-recoverability — O0→R0 content-category pre-registration

This protocol is a mechanism-only child of [[spec-reconstruction-bench]]. It asks which kinds of intent in
the natural C0 O0 can be completely recovered by a reconstruction. It does **not** replace or modify the
parent benchmark's primary estimand: forward decision/implementation utility on real post-C0 episodes.
No future-utility result is pooled with this study.

Frozen ancestry is inherited without reinterpretation: `C_eval=5723eaca1ba52eb60d28b818ddb4f77d758af01a`,
calendar time-split `C0=038dce1f7e87b5ef3c8737daffb4fe929d1753a7`, parent `targets.json`, the allowlist-default-deny snapshot,
forbidden surfaces, masked-shingle scan, future-line canary, and clean/leak-positive twin. Any later R0 must
pass the parent's dry snapshot gates at the same frozen bytes. This round launches no constructor or judge,
makes no network call, spends nothing, reads no archived leaf R0, and creates no R0. Effective module R0 is 0.

## Estimand and scales

For each content category, the primary mechanism estimand is the module-cluster macro description of
`P(judge verdict = full | O0 unit is A-supported, category, module scale)` under the frozen constructor and
judge protocol. `partial` is a lenient secondary diagnostic, not success. Unsupported precision, fabricated
false recovery, B-latent recovery, and recovered S are separate diagnostics.

- **module primary** — reconstruct an eligible finite parent subtree or one member of a size-matched package
  group. Output must carry module responsibility and non-responsibility, node topology and ownership, and
  cross-leaf contracts, not merely concatenate leaf prose.
- **leaf calibration** — the parent's two archived leaf targets (`spec-lint`, `mobile-ui`) may be reused only
  after all sample annotations are frozen. Their tracked-repository budget differs from the module code-only
  budget, so they remain calibration and are never pooled with modules.
- **whole excluded** — no whole reconstruction, whole score, scale curve, or whole inference exists here.

## External module frame and census

Eligibility is computed only from the C0 tree plus the already-frozen package roster in parent
`targets.json`; O0 body content, R0, future episodes, and outcomes cannot affect it.

1. A parent-subtree candidate is a non-hidden node strictly below a rostered product-package root, with
   2–8 descendant nodes and maximum descendant depth ≤2 relative to the candidate root.
2. A size-matched package candidate is a rostered package root satisfying the same descendant/depth bounds
   and having at least one other rostered package root with the exact same descendant count. Each package is
   an analysis cluster; the match is the external eligibility condition, not a pooled pair.
3. Eligible clusters must not overlap. Every eligible cluster enters the module census; there is no module
   sampling. The deterministic launch order is only scheduling and never changes the census.

At frozen C0 these rules produce six modules: `sessions/injected-context` (3 nodes total),
`sessions/lifecycle` (4), `sessions/comms` (5), `source-of-truth` (9: root plus 8 descendants),
`spec-forge` (7), and `spec-yatsu` (7). This exactly fixes the later budget surface at six module
reconstructions plus two reused leaves; it is not a post-hoc sample.
If recomputation yields fewer than four eligible modules, module-primary is NO-GO and only leaf calibration
may proceed. A result with `n≤6` is a descriptive protocol pilot.

The earlier five-module capture was wrong: it counted the root inside the `2–8` bound and therefore dropped
`source-of-truth`; the frozen rule counts descendants/children, as the critique states, so its eight direct
children make it eligible. Hidden `.config` is excluded by the hidden-path rule before size is considered;
the visible `config` system-spec node is outside the frozen product-package roster; `session-console` has only
one child. None can enter as a plugin-shaped or one-child pseudo-module. `frame.json` freezes these reasons.

For every candidate, including exclusions, `frame.json` records candidate kind, node/descendant/direct-child
counts, maximum depth, referenced file count, text LOC, missing references, body characters, inbound and
outbound wiki edges, external related references, cross-module coupling (the sum of those three edge counts),
and breadcrumb density (`inbound edges / node count`). The exact definitions and every exclusion reason are
generated by the runner and frozen before any R0.

## Deterministic O0 unit census

`seg-v2` reads only C0 `spec.md` bodies. It drops headings as navigation, excludes fenced code as non-prose
while recording the excluded block count, and applies the same sentence boundary to bullets and paragraphs:
Chinese/English `.?!。！？` ends a unit, while semicolons remain inside one commitment group. Bullet
continuations are joined before splitting. A Markdown table header is context, not a unit; each data row is
rendered as a self-contained `header: value` statement. A clause beginning with a conjunction or lowercase
continuation is attached to the preceding unit (or the following unit at file start), so era prose
such as a trailing parenthetical is not promoted into a fake commitment. No annotator or constructor may
redraw boundaries.

Every unit from every node in every eligible module enters `units-skeleton.json`; no unit is sampled.
The two leaf targets are segmented by the same code for calibration. Unit ids bind seed, cluster, source
node, ordinal, and raw text. Each source node also receives a seeded anonymous `component-NN` slot inside its
cluster; statements and candidate bundles use the slot rather than a node id, so pronouns and ownership stay
independently judgeable without unblinding the source. Shuffled-original permutes these exact slots. Counts
outside the suggested 60–120/module or 20–30/leaf are reported as a diagnostic, never trimmed.

The same segmenter runs on every node in the frozen non-sample taxonomy dev pool. `segmentation-audit.json`
freezes those units and rejects headings, raw table syntax, dependent-start fragments, units under 20 chars,
or units over 600 chars. This is a grain check, not a unit sampler. It also records the actual sample-unit
count and the full dual-annotation/fabricated-verification budget; a count outside the suggested range must
be explained here from the measured distribution before reconstruction, never silently trimmed.

The frozen result is 620 module units across six modules: 100 `comms`, 34 `injected-context`, 73 `lifecycle`,
166 `source-of-truth`, 122 `spec-forge`, and 125 `spec-yatsu`. This is far above the critique's provisional
~60–120 module-scale pilot estimate; mean/median per module do not repair that mismatch. The smallest allowed
2-descendant module is genuinely small, while the allowed 8-child and seven-node package modules carry many
independent commitments; the single external rule and zero-unit-sampling constraint keep all of them. Full
review of the non-sample output and the mechanical audit found no heading, raw
table row, dependent-start fragment, <20-char unit, or >600-char unit; `segmentation-audit.md` records that
review. The actual pre-R0 burden is 659 real cards (620 module + 39 leaf), 1,318 independent real labels, 95
fabricated units with 190 independent absence verifications: at least 1,508 independent decisions before as
many as 659 adjudications. This is not a minimal or quick pilot. It creates an annotation-feasibility NO-GO:
before any R0, a human must explicitly approve this exact full-census burden, or the frame must be re-frozen
under a new external, pre-registered module-scope rule while R0/results remain unseen. Unit sampling is never
an allowed repair.

## Taxonomy and legacy-language rewrite

Before sample labels, the seven-category taxonomy is developed and frozen on a deterministic non-sample C0
dev pool (eligible-module and leaf-calibration nodes, plus their ancestors, excluded). Each unit receives exactly one primary label
using the precedence in `taxonomy.json`: historical incident, design rationale, ownership topology,
responsibility boundary, constraint invariant, operational mechanism, then behavioral contract. Secondary
tags may aid audit but never enter primary aggregation. `historical-incident` remains secondary-only even if
its cell later meets a numeric minimum.

The two raw primary-category labels are retained. Cohen's kappa must be ≥0.6. One taxonomy clarification and
complete blind relabel is allowed; if kappa remains below 0.6, the study is NO-GO. The revision rewrites the
living taxonomy and requires a new freeze; it is not an append-only exception.

C0 predates the yatsu→eval vocabulary migration. Judges therefore see a behaviorally neutral rewrite, not
the raw era label. `rewrite-map.json` binds each raw hash to its rewritten statement and applied rule:
`spec-yatsu`, `yatsu-core`, `yatsu.md`, `yatsu.evals.ndjson`, `needs-yatsu-eval`, and bare `yatsu` become
neutral evaluation/measurement phrases; `.config` becomes the plugin-configuration surface. The runner
rejects residual legacy tokens and rejects normalization collisions between distinct raw units. Raw bytes
remain archived only in the sealed map for audit. Any semantic manual rewrite changes the map and requires
a new byte freeze before annotation.

## Pre-R0 A/B/S dual labels

Two independent annotators label 100% of sample units before any module R0 exists. They may see the O0 unit
and clean C0 code-only snapshot, but no R0, future episode, parent leaf R0 archive, arm label, or other
annotator's decision.

- **A-supported** — the allowed C0 artifacts independently support the complete statement. Each label cites
  at least one C0 path/line evidence reference.
- **B-latent** — O0 states the intent, but allowed C0 artifacts neither independently support nor contradict
  it. It is historical/latent knowledge, not reconstruction truth.
- **S-stale-contradicted** — the C0 artifacts contradict the statement or show that it was stale at C0.

Raw labels remain in the card. A blind adjudicator resolves disagreements before freeze. The main recovery
table uses only adjudicated A units. B full recovery above 20% halts interpretation for a contamination
audit; it never upgrades B into truth. Every recovered S is a separately listed bad signal.

`labels --write` validates one card per skeleton unit, two distinct annotators, closed keys, evidence for A,
taxonomy membership, fabricated controls, and kappa; it then writes an exact-byte annotation manifest.
`labels --check` revalidates and hashes the same bytes. Both must succeed before reconstruction preflight.
Annotators attest `moduleR0AtFreeze=0` and `phaseLeafR0ReadBeforeFreeze=false`.

## Reconstruction knowledge and leakage discipline

Module R0 uses the parent's `production-code-only` snapshot surface. It inherits the exact C0/time-split,
allowlist default-deny, forbidden paths, clean masked-shingle scan, future-line canary, and leak-positive twin.
The parent dry oracle must be rerun at the frozen runner/targets/protocol hashes immediately before a paid
phase. A clean snapshot hit, a twin that fails to alarm, a prompt hit, a symlink/git residue, or any R0 phrase
overlap violation invalidates the batch. This child protocol cannot weaken those gates.

The module constructor receives no O0, tree labels, unit cards, taxonomy, rewrites, controls, future data, or
leaf archive. Its neutral output contract requires component slots, responsibility/non-responsibility,
topology/ownership, cross-leaf contracts, and prose intent. Failure or an invalid output is not N0 and is not
silently scored. Constructor and all later paid work remain behind the parent's reviewer authorization,
executor identity, isolation, usage, secret-scan, provenance, and archive gates. This runner deliberately has
no model adapter and no network API.

## Blind controls and breadcrumb ablation

Each module receives the same frozen control surface:

- **O0-self positive** — correctly normalized O0 in its original anonymous ownership slots. Full recall
  below 95% is NO-GO because the rubric cannot recover its own positive truth.
- **distractor module** — an intact deterministic derangement to another census module; no cherry-picking by
  semantic similarity or result.
- **shuffled original** — target O0 statements are deterministically reassigned across wrong node/ownership
  slots while slot counts and style remain. It tests whether judges notice responsibility/topology rather
  than rewarding familiar prose.
- **fabricated plausible units** — each scale's complete census receives the integer count nearest 12.5%
  while remaining inside 10–15%; slots are then allocated across its clusters by deterministic largest
  remainder, so a small leaf is not forced into an impossible per-cluster fraction. Two blind verifiers
  confirm each is plausible yet unsupported and
  uncontradicted in C0. False `full` recovery above 10% is NO-GO.
- **tree-only ablation** — zero-paid deterministic reconstruction from the retained external tree and inbound
  breadcrumbs, with no code or masked subtree. It is judged on topology/ownership units through the same
  packets. If its full count/rate is at least 80% of R0's, scrub/stub inbound ids and rerun that comparison or
  demote topology/ownership to secondary. Breadcrumb density is reported module by module.

`control-plan.json` freezes distractor pairings, shuffled-slot seed, scale-level fabricated rates and their
deterministic cluster slot allocation, and tree-only
surface before R0. Fabricated text and tree-only judgments are separately human-authored then included in
the annotation freeze. No control identity appears in blind packet bytes.

`tree-only-ablation.json` is the frozen zero-paid output itself: for each masked module it contains only the
retained tree paths, external inbound mention tuples, and the component-existence/target-ownership claims
mechanically entailed by those tuples. It contains no code, masked body, R0, or model output. The later
`tree-only-results.json` applies frozen unit labels/rubric to this output; that result cannot exist honestly
until dual labels exist, so it remains a preflight requirement rather than a fabricated green score here.

## Packet blinding, style, and judges

Statements use rewritten text. Candidate order is seed-shuffled. Packets have closed keys and contain no arm,
control, category, stratum, node path/id, constructor, source, or seed. The unblinding map lives outside the
packet tree; only its SHA-256 commitment enters the manifest. Fabricated probes have identical packet shape.

Before content judging, the panel performs a held-out style identity probe. Accuracy >70% triggers a frozen
arm-neutral normalization pass and a new probe. A second >70% result is NO-GO. Normalization may remove style
but not behavior, ownership, topology, or qualifiers.

The panel is at least one human plus two model judges from different families. Constructor family
OpenAI/gpt-5.5 is excluded from model judges. Every judge uses the frozen four-verdict rubric. Reports contain
each judge separately, consensus, Gwet AC2, Krippendorff alpha, and leave-one-judge-out (LOO). If removing any
judge reverses category ordering, no consensus ordering or pooled consensus claim is reported.

## Aggregation and hard reporting limits

Primary success is `full`; `partial` never enters it. The unit is scored inside its module, then module rates
are macro-described by category. Leaf calibration is reported separately. No module/leaf pooling, micro-
average, composite, overall score, scale curve, or whole result is permitted.

A category×scale cell needs at least 3 clusters and 15 A-supported units. Below either bound, output contains
only counts plus `insufficient`; it contains no rate or confidence interval. At the actual `n≤6`, sufficient
cells may show per-cluster full/n and a descriptive macro mean/min/median/max, but no bootstrap, CI,
significance test, non-inferiority, pooled score, or scale curve. Historical incidents are secondary-only.
Category marginals are descriptive; category×A/B/S crossings remain visible.

Controls are gates, not outcomes: O0-self <95% or fabricated false recovery >10% is NO-GO; B >20% requires
contamination audit before any content conclusion; recovered S is bad signal; tree-only ≥80% of R0 forces
scrub/stub or secondary status; style >70%, category kappa <0.6 after one revision, constructor-family judge,
or LOO category-order reversal blocks the corresponding consensus claim.

## Freeze order, current state, and later budget

The checked-in zero-model runner has three boundaries:

1. `node run.mjs --write` deterministically writes `frame.json`, `units-skeleton.json`,
   `segmentation-audit.json`, `rewrite-map.json`, `control-plan.json`, `tree-only-ablation.json`,
   `run-state.json`, and `freeze-manifest.json`; `--check` recomputes and byte-compares
   every generated file and verifies exact hashes of protocol, critiques, schema, taxonomy, template, rubric,
   parent targets, parent protocol, and parent runner.
2. After a human explicitly approves the exact census burden, real dual annotation but before R0, `labels
   --write` validates and freezes `annotation-feasibility-approval.json`, `unit-cards.json`,
   `fabricated-units.json`, `tree-only-results.json`, `style-probe.json`, and `judge-panel.json`; `labels
   --check` byte-verifies them. These files do not exist in this pre-registration commit, so preflight must
   fail loudly rather than pretend approval or labels exist. Approval must name 620 module units, 659 real
   cards, and at least 1,508 independent decisions, and bind the current protocol freeze hash. The alternative
   is a new external module-scope rule and complete pre-R0 frame re-freeze, never unit sampling.
3. `preflight` requires both freezes, parent dry/canary evidence, all human attestations, current module R0=0,
   and explicit later reviewer authorization. It launches nothing. The actual constructor remains in a later
   separately authorized phase outside this runner.

`dry --case positive` admits a synthetic valid fixture. `dry --case negative` returns nonzero only after
exercising packet leakage, O0-self, fabricated, B, S, kappa, style, breadcrumb, constructor-family, LOO, and
forbidden aggregation gates. Plain `dry` succeeds only when the positive fixture is accepted and the
negative fixture is rejected. Exact stdout/stderr and each independent rc are frozen as eval evidence.

The later authorized pilot budget is exactly the six-module census plus two reused leaves. No extra module
may be selected to repair a bad result, and no eligible module may be dropped. At this commit: paid runs 0,
network calls 0, generated module R0 0, consumed module R0 0, archived leaf R0 reads by annotators 0.
